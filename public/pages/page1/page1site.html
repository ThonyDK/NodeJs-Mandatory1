SSR and CSR: To make the app run as ssr then we need to add a javascript file with the alias templateEngine. This peace of code will be the keyholder to render pages and in the same time also exclude redundancy of code. For example having a navbar rendering from the navbar directory instead of having navbar on multiple html sites. inside the templateEngine there will be used a method called fs. Fs is a importet method that makes it possible to readFile like html pages. The templateEngine that holds the renderpage function will make sure that the right html page will come to the client and with the right css styling to the page.
Client side rendering also called csr is another way to make the application running. Client-side rendering means that a websites JavaScript is rendered in your browser, rather than on the websites server. So now, instead of getting all the content from the HTML doc, only the required HTML with the JS files will be rendered. So the question is why use ssr vs csr?: Server-side rendering is better for SEO than client-side rendering. This is because server-side rendering can speed up page load times, which not only improves the user experience, but can help your site rank to be better in Google search results.
